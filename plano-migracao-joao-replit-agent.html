Plano de Migração — João para Replit Agent

Recomendação principal

Objetivo: tirar o João da dependência operacional do Lovable na VPS atual e colocá-lo em um ambiente mais estável para criação, iteração e continuidade de projetos.

1. Decisão executiva

Ponto Decisão Motivo
Plataforma principal Replit Agent Maior chance de operação contínua com menos fragilidade de sessão.
Plataforma de contingência Bolt.new Serve como backup mais próximo da experiência de geração por prompt.
Uso do v0 Apoio visual Excelente para UI, mas não como base principal do João.

2. Escopo inicial do João no Replit

  • Criar landing pages e páginas institucionais do IVS.
  • Prototipar painéis internos e fluxos operacionais.
  • Iterar em projeto vivo sem depender de sessão frágil em browser externo.
  • Manter histórico de evolução e continuidade técnica.

3. Plano em 4 fases

Fase 1 — Preparação

  • Criar ou validar conta institucional que o João vai usar.
  • Definir se o acesso será individual ou por credencial operacional dedicada.
  • Escolher 1 projeto piloto do IVS para teste.

Fase 2 — Prova operacional

  • Subir um projeto teste com briefing real do IVS.
  • Validar criação, edição, preview e continuidade.
  • Medir se o João consegue operar sem intervenção manual frequente.

Fase 3 — Padronização

  • Criar prompt-base canônico do João para projetos web.
  • Definir stack padrão: estrutura, UI, integração e publicação.
  • Documentar processo no cérebro do IVS.

Fase 4 — Produção

  • Migrar os pedidos novos de construção/protótipo para o ambiente homologado.
  • Manter Lovable apenas como contingência ou frente paralela.
  • Estabelecer checklist de QA antes de qualquer entrega interna.

4. Critérios de sucesso

Critério Meta mínima
Criação de projeto Funcionando sem bloqueio de sessão
Edição contínua 3 rodadas de iteração sem reset manual
Preview Renderização estável
Autonomia do João Baixa dependência de intervenção humana

5. Riscos e mitigação

Risco Nível Mitigação
Nova plataforma não atender ao estilo de geração esperado Médio Rodar projeto piloto curto antes de migração maior.
Curva de adaptação do João Médio Padronizar prompts e processo operacional.
Dependência futura de outra UI web Médio Escolher ambiente com melhor continuidade e fallback definido.

6. Recomendação final

Recomendação: abrir imediatamente uma prova operacional do João no Replit Agent com um projeto piloto do IVS e usar o Bolt.new como comparação de backup.

Leitura gerencial: isso reduz o tempo perdido insistindo na VPS atual do Lovable e nos leva para uma trilha mais provável de operação estável.

7. Próximo passo sugerido

Montar agora um briefing piloto do IVS para testar no Replit Agent e no Bolt, com o mesmo escopo, para decisão rápida.
