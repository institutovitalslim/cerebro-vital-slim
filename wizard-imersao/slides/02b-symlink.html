🔗 Symlink — como o agente se conecta ao Cérebro

Em vez de copiar arquivos, o workspace do agente aponta direto pro repositório. Mesmo arquivo, sempre sincronizado.

❌ Sem symlink (cópia)
📄
SOUL.md
workspace/SOUL.md
cópia
≠
📄
SOUL.md
cerebro/agentes/marketing/SOUL.md
cópia
⚠️ Atualiza um, o outro fica desatualizado
✅ Com symlink (atalho)
🔗
SOUL.md
workspace/SOUL.md → aponta pro repo
symlink
═
📄
SOUL.md
cerebro/agentes/marketing/SOUL.md
original
✅ Mesmo arquivo. Atualizar um = atualizar o outro
🔄
Quando o agente lê o SOUL.md, ele tá lendo direto do cérebro no repo. A cada alteração, o agente faz push pro GitHub — o cérebro é sempre a versão mais atual. Sem duplicação, sem risco de ficar fora de sync.
