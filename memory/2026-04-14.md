## 2026-04-14 — Tweet-carrossel v3: bloqueios reais e caminho validado

### Google / NanoBanana 2
- Não confiar no `image_generate` sozinho para afirmar provider usado, porque ele pode cair em fallback silencioso para OpenAI
- Validar sempre o provider real no retorno ou testar direto via SDK/API do Google antes de afirmar “foi NanoBanana 2”
- `gemini-3.1-flash-lite-preview` pode falhar com `503 UNAVAILABLE` por alta demanda
- `gemini-3.1-flash-image-preview` pode falhar com `429 RESOURCE_EXHAUSTED` (quota exceeded / limit 0)
- Se isso acontecer, tratar como bloqueio real e não entregar capa como se estivesse no fluxo Google

### Chromium headless
- Mesmo com o browser tool indisponível, `google-chrome --headless=new --disable-gpu --no-sandbox --hide-scrollbars --window-size=... --screenshot=... URL` funcionou para capturar PubMed e renderizar HTML local
- Isso viabiliza o slide 2 no formato tweet + screenshot PubMed sem depender do browser MCP

### Recuperação de copy aprovada
- A copy interna aprovada do carrossel de creatina ficou preservada em `deliverables/carrosseis-aprovados-2026-04-13/creatina/slides.json`
- Em caso de falha da memória indexada, esse arquivo serve como fonte operacional confiável para regenerar slides 3+

### Entrega parcial útil
- Pasta criada: `deliverables/tweet-carrossel-v3-2026-04-14/`
