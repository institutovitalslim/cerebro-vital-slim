## Bridge HTTPS do OpenClaw montada

- Problema: Claude rodando fora da VPS (sandbox com HTTPS-only outbound) precisava acesso de monitoring ao gateway local `localhost:18789`.
- Solução escolhida: **nginx + Let's Encrypt + bearer auth** na VPS, subdomínio próprio. DNS na HostGator, A record `openclaw.institutovitalslim.com.br` → `187.77.58.193`.
- Scripts: `ops/setup/openclaw-bridge.sh` (install idempotente) e `ops/setup/rotate-bridge-token.sh` (rotação).
- **Princípio do menor privilégio aplicado:** o nginx valida o `BRIDGE_TOKEN` externo e faz proxy pro gateway interno **sem** repassar o header Authorization — ou seja, endpoints que exigem `gateway.auth.token` do OpenClaw retornam 401 via bridge. Isso limita o estrago de um eventual vazamento do BRIDGE_TOKEN: atacante vê `/health`, `/version`, UI pública, mas não dispara skills, não manda WhatsApp, não mexe em paciente.
- Após montar, rotacionei o token (estava exposto no transcrito de chat) via `rotate-bridge-token.sh`. Token antigo confirmado como revogado (401).
- URL: `https://openclaw.institutovitalslim.com.br` — liveness em `/__bridge_alive`.

## Problemas interessantes encontrados no setup do bridge

1. **Ubuntu 25.10 apt silent failure:** `apt-get install -y -qq nginx ... >/dev/null` retornou exit 0 sem instalar nginx. Reinstalar com `nginx-core` direto funcionou. Hipótese: bug conhecido no indexador do apt nessa versão (mesmo bug que quebra updates automáticos — ver `discourse.ubuntu.com/t/enabling-updates-on-ubuntu-25-10-systems/`).
2. **hpanel browser terminal PATH restrito:** o shell via console web da Hostinger (`cam.hostingervps.com`) não tem `/sbin` nem `/usr/sbin` no PATH, mesmo como root. `nginx`, `systemctl`, `certbot` ficam indisponíveis por nome. Fix no script: `export PATH="/usr/local/sbin:/usr/local/bin:/sbin:/usr/sbin:/bin:/usr/bin:${PATH:-}"`.
3. **nginx map_hash_bucket_size insuficiente:** bearer token de 64 chars hex + prefixo "Bearer " = 71 chars excede o default (64) do `map_hash_bucket_size`. Fix: `map_hash_bucket_size 128; map_hash_max_size 2048;` no topo do site config.
4. **iOS Smart Punctuation + Hostinger terminal:** qualquer URL digitada via teclado iOS ou pelo terminal web da Hostinger é envelopada em `<...>`. Bash interpreta `<` como redirecionamento. Solução: usar git no lugar de curl/wget quando possível; alternativamente desativar Pontuação Inteligente do iOS.
5. **Sandbox do Claude bloqueia TCP arbitrário:** só HTTP/HTTPS saem. Porta 22, 53, etc., tudo bloqueado. É o motivo pra montar a bridge HTTPS em vez de usar SSH direto.
