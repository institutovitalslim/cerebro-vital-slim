# Output templates

> For the **designed PDF report** — the recommended, presentable format — see
> `references/designed-pdf.md`. The templates below are the content backbone that
> feeds every format (PDF, chat, docx, xlsx); the PDF wraps this same content in
> the branded visual layout.

Two templates: the per-competitor brief and the cross-competitor synthesis.
These are guides, not rigid forms — adapt headings to what the research actually
found. The goal is something a reader can scan in 30 seconds and trust.

## Per-competitor brief

```
## [handle / name] — [real name or brand, if found]
[One line of identity: who they are and the lane they're in. Note any handle
correction or disambiguation here.]

- ICP: [Who they sell to. Separate audience from paying customer if they differ.
  Name the pain points.]
- Funnel: [Top → middle → bottom in one flowing line. Note missing stages.]
- Money: [Every revenue stream with prices. Lead with the main engine.]
- Numbers: [2–3 metrics with source + date — e.g. followers + growth, traffic,
  engagement, subs. Tag CONFIRMED / REPORTED / INFERRED.]
- Links: [Their own properties as clickable links — site · Instagram · TikTok ·
  X · YouTube · LinkedIn · newsletter · podcast · Linktree/Stan · community/store.
  Only the ones that actually exist.]
```

Always include the Links directory — it's the map of the competitor's funnel and
the thing users most often want to click into. Keep it separate from Sources:
**Links = the competitor's own pages; Sources = where the facts came from.**

Keep each brief to a few lines. If something is inferred rather than confirmed,
say so inline ("likely…", "unverified claim from their own bio"). End the full
set of briefs with the sources, or keep sources per-brief if there are only a
few competitors.

## Cross-competitor synthesis

Lead with what they have in common, then where they split, then a maturity
ranking, then the one-line takeaway.

```
## Synthesis

Who they are: [One line placing all of them in the same/different lanes.]

ICP patterns: [Where targeting converges and diverges.]

Funnel patterns: [The shared mechanics up top, the divergence at the offer.]

Threat ranking — by built-out monetization AND size/momentum (use the numbers):
1. [Biggest threat] — [why: scale + how-built-out, citing the metrics]
2. ...
N. [Least / earliest-stage] — [why]

Sharpest takeaway: [One line the user should remember.]

Gaps: [Anything that couldn't be verified, stated honestly.]
```

## Word doc (if chosen)

Read the `docx` skill first. Structure: title page / heading, one section per
competitor (using the brief template as the section body), then the synthesis,
then a sources appendix. Keep it clean and readable — this is the version people
hand to others, so it should look like a real research deliverable.

## Spreadsheet matrix (if chosen)

Read the `xlsx` skill first. One row per competitor. Suggested columns:

| Competitor | Real name / brand | ICP (audience) | ICP (buyer) | Top of funnel | Lead magnet | Core offer | Pricing | Other revenue | Size/traffic | Growth | Key links (site, socials, podcast) | Confidence |

Add a final row or a second sheet for the synthesis notes. A matrix is the best
format when there are many competitors and the user wants to compare at a glance.
