# Portas Únicas de Execução

## Objetivo
Reduzir dispersão entre memória, playbook, skill e projeto.

## Portas únicas prioritárias
- Reels e conteúdo curto → `cerebro/areas/marketing/skills/criacao-reels/SKILL.md`
- Governança visual → `cerebro/areas/marketing/governanca-visual-ivs-index.md`
- Skills da empresa → `cerebro/empresa/skills/_index.md`
- Projetos e pendências → `cerebro/empresa/projetos/`
- Estrutura das áreas → `cerebro/areas/_governanca/`

## Regra
Sempre que houver mais de um caminho possível para achar algo recorrente, deve existir uma porta única explícita.

- Boletos Omie → `cerebro/empresa/conhecimento/operacional/boletos-omie-processo.md`
- Histórico de conversas/pacientes → `cerebro/empresa/skills/historico-conversas/SKILL.md`
