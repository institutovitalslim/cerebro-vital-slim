# Cross-Topic Protocol

Use este protocolo quando estiver em um grupo Telegram com múltiplos tópicos.

## Regra principal
Não presumir que o tópico atual contém todo o contexto relevante.
Toda informação importante descoberta em um tópico deve ser promovida para a memória canônica geral, para que execuções futuras não dependam do contexto local daquele tópico.

**Regra reforçada por Tiaro em 2026-04-21:** todos os tópicos do mesmo grupo devem funcionar como partes de **um conhecimento único**. O tópico é só a interface da conversa; a memória e o cérebro precisam permanecer unificados.

## Fluxo obrigatório
1. Identificar o domínio da solicitação.
2. Consultar `cerebro/OPERATIONS_INDEX.md`.
3. Consultar o arquivo canônico do domínio.
4. Consultar `cerebro/telegram-topics.md` para entender a origem e distribuição do contexto entre tópicos.
5. Se a decisão anterior tiver surgido em outro tópico, usar a memória canônica promovida daquele domínio.
6. Se surgir nova regra, decisão, convenção ou contexto recorrente, promover isso imediatamente para a memória canônica adequada.
7. Responder ou executar apenas depois dessa checagem.

## Quando promover contexto entre tópicos
Promover imediatamente quando o conteúdo envolver:
- regra operacional
- decisão recorrente
- contato importante
- convenção interna
- fluxo técnico
- IDs / rotas / agendas / integrações
- significados específicos definidos por Tiaro

## Regra de ouro
O tópico é um ponto de conversa; a memória de verdade deve viver na estrutura canônica transversal.
Nunca depender só do contexto do tópico para executar uma tarefa recorrente.
A resposta correta em qualquer tópico deve refletir o melhor conhecimento consolidado do grupo inteiro, e não apenas do histórico local daquele tópico.
