# Learning Protocol

Este protocolo define como novas aprendizagens devem ser incorporadas ao cérebro.

## Objetivo
Evitar repetição de erros, perda de contexto útil e dependência de memória improvisada.

## Regra principal
Sempre que surgir um novo aprendizado relevante — seja por correção do usuário, descoberta operacional, erro, decisão, convenção ou detalhe recorrente — registrar e incorporar esse aprendizado no mesmo fluxo de trabalho.

## Classificação
### Nível 1 — Canônico
Use quando o aprendizado for recorrente, crítico ou estrutural.
Ação:
- atualizar um arquivo canônico existente; ou
- criar um novo arquivo canônico se o domínio ainda não existir;
- atualizar `cerebro/OPERATIONS_INDEX.md`;
- registrar em `cerebro/learning-ledger.md`.

### Nível 2 — Operacional
Use para IDs, fluxos, contatos, endpoints, convenções e regras práticas recorrentes.
Ação:
- atualizar o arquivo do domínio;
- atualizar o índice se necessário;
- registrar no ledger.

### Nível 3 — Diário
Use para fatos contextuais úteis, mas sem recorrência estrutural.
Ação:
- registrar na memória diária;
- opcionalmente registrar no ledger se o aprendizado puder evoluir depois.

### Nível 4 — Descartável
Use quando não houver valor recorrente claro.
Ação:
- não criar novo arquivo;
- não poluir o índice.

## Fluxo obrigatório pós-aprendizado
1. Identificar o aprendizado.
2. Classificar o nível.
3. Atualizar o arquivo canônico ou operacional correto.
4. Atualizar `cerebro/OPERATIONS_INDEX.md` se necessário.
5. Registrar a mudança em `cerebro/learning-ledger.md`.
6. Se a mudança for estrutural, fazer commit no cérebro.

## Regra anti-caos
- Não criar arquivo novo sem necessidade real.
- Preferir atualizar arquivos de domínio existentes.
- Criar arquivo novo apenas quando um domínio novo ou uma categoria nova surgir de forma recorrente.
- Índice deve permanecer curto e navegável.
